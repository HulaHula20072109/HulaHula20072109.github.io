超文本传输协议（Hypertext Transfer Protocol）是一个用于传输超媒体文档（例如 HTML）的应用层协议。它是为 Web 浏览器与 Web 服务器之间的通信而设计的，但也可以用于其他目的。HTTP 遵循经典的客户端—服务端模型，客户端打开一个连接以发出请求，然后等待直到收到服务器端响应。HTTP 是无状态协议，这意味着服务器不会在两个请求之间保留任何数据（状态）。

详情：MDN

URI 与 URL 与 URN

统一资源标识符 URI

在 Http 请求中会包含多种类型的 资源，为了正确地访问到确定位置的资源，我们使用 URI (Uniform Resource Identifier) 统一资源标识符来定位物理资源，以便请求访问。它是一个概念，并不限制具体表现格式，只是规定了是一种字符串

常见的 URI 包括 URL 和 URN 两种形式

统一资源定位符 URL

URI 的最常见形式是统一资源定位符 (Uniform Resource Locator)，也被称为 Web 地址，它是对可以从互联网上得到的资源的位置和访问方法的一种简洁的表示，是互联网上标准资源的地址。

字段 含义
scheme 有我们很熟悉的http、https、ftp以及著名的ed2k，迅雷的thunder等。
host HTTP服务器的IP地址或者域名
port HTTP服务器的默认端口是80，这种情况下端口号可以省略。如果使用了别的端口，必须指明，例如tomcat的默认端口是8080
path 访问资源的路径
url-params 所带参数
query-string 发送给http服务器的数据
anchor 锚点定位
