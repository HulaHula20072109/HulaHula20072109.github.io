Http消息主要分为两种：请求和响应

请求头

响应头

http请求和响应头均包含了大量字段，这些字段可以在相关文档中查询意义

Http消息结构

无论哪种消息类型，均包含起始行、标头以及主体三部分

Http请求

起始行

起始行就是第一行的字符串，包含方法、请求目标、版本

  1. 方法表示对资源的操作，常见有POST、GET，详见Http请求方法
  2. 请求目标，通常为url，详见HTTP协议 | URI
  3. 版本定义了剩余消息的结构，不同版本的数据格式有出入。1997 年 1 月公布的 HTTP/1.1 是目前主流的 HTTP 协议版本。 当初的标准是 RFC2068，之后发布的修订版 RFC2616 就是当前的最新版本。

标头

标头由一系列键值对组成
具体意义详见 HTTP标头

主体

不是所有的请求都有一个主体：例如获取资源的请求，像 GET、HEAD、DELETE 和 OPTIONS，通常它们不需要主体。有些请求将数据发送到服务器以便更新数据：常见的的情况是 POST 请求（包含 HTML 表单数据）。

Http响应

起始行

Http响应的起始行也包含三个内容

  1. 协议版本，通常为 HTTP/1.1
  2. 状态码（status code），表明请求是成功或失败。常见的状态码是 200、404 或 302
  3. 状态文本（status text）。一个简短的状态码的文本描述
状态码 意义
200 请求成功 OK
302 重定向 Found
403 禁止请求 Forbidden
404 找不到资源 Not Found
502 网关上游发生错误 Bad Gateway

更多状态码参见 HTTP响应状态码

标头

响应头记录了服务器相关的信息，以及主体内容格式和字符集 (Content-Type)，与请求头相对应

主体

响应的具体数据信息，例如一个html/css/js文件、图片、视频等等
